## 1. 개요

SKT 아이콘 시스템은 **2D Icon**과 **Line Icon** 두 가지 스타일로 구성됩니다. 각 스타일은 서로 다른 사용 맥락과 표현 방식을 가지며, 브랜드 일체감을 유지하면서 효과적인 정보 전달을 목적으로 합니다.

| 구분 | 2D Icon | Line Icon |
|------|---------|-----------|
| **스타일** | 입체감 있는 일러스트레이션 | 단색 아웃라인 스트로크 |
| **수량** | 44종 (2개 세트: 범용 10종 + 커뮤니케이션 34종) | 191종 (12개 카테고리) |
| **주요 용도** | 프로모션, 마케팅, 서비스 커뮤니케이션 | UI, 서비스 내비게이션, 기능 아이콘 |
| **제공 포맷** | AI, PDF, PNG | AI, PDF, PNG |
| **배경** | 투명 배경 제공, 다크 배경 위 배치 권장 | 투명 배경 제공, 라이트/다크 모두 가능 |

## 2. Line Icon — 디자인 원칙

### 2.1 디자인 철학

라인 아이콘은 **대내외적으로 브랜드 일체감을 조성하고 효과적인 정보 전달이 가능하게** 합니다. T-B 로고의 직선, 곡선적 형태 특징과 사선을 활용하여 제작되었습니다.

핵심 조형 원칙 4가지:

- **Straight Shape** — 직선 기반의 깔끔한 형태
- **Diagonal Shape** — T-B 로고에서 파생된 사선(120°) 활용
- **Round Corner** — 꼭짓점에 라운드 처리
- **1 Line Stroke** — 단일 굵기의 일관된 스트로크

### 2.2 그리드 시스템

```
┌──────────────────────────┐
│  Padding Area (2dp)      │
│  ┌──────────────────┐    │
│  │                  │    │
│  │   Design Area    │    │
│  │   16dp × 16dp    │    │
│  │                  │    │
│  └──────────────────┘    │
│                          │
└──────────────────────────┘
       Total Area
      20dp × 20dp
```

| 영역 | 크기 | 설명 |
|------|------|------|
| **Total Area** | 20dp × 20dp | 아이콘의 사이즈를 정의하는 전체 영역. Design Area와 Padding Area로 구성 |
| **Padding Area** | 상하좌우 2dp | 아이콘 영역을 둘러싸는 여백 공간. **반드시 확보해야 함** |
| **Design Area** | 16dp × 16dp | 기본적으로 정방형의 비율을 유지하되, 실제 사물의 비율을 표현해야 하는 경우 타 픽토그램과 균일한 크기감을 갖도록 조정하여 사용 |

### 2.3 스트로크 규격

| 속성 | 값 | 비고 |
|------|------|------|
| **스트로크 굵기** | 0.85dp | 전체 아이콘 동일 적용 |
| **사선 각도** | 120° | T-B 로고 사선에서 파생 |
| **커팅 시 사이 공간** | 굵기와 동일 여백 적용 | 선이 끊어지는 부분의 간격 = 스트로크 굵기 |
| **색상** | 단색 (기본 #040000) | 거의 블랙에 가까운 단색, RGBA 모드 |
| **라운드 코너** | 적용 | 모든 꼭짓점에 라운드 처리 |

### 2.4 최소 사이즈

| 매체 | 최소 사이즈 | 기준 |
|------|------------|------|
| **인쇄** | 20mm | Total Area 기준 |
| **화면(스크린)** | 48px | Total Area 기준 |

> ⚠️ 최소 사이즈 이하로 사용 시 스트로크가 뭉개지거나 판독이 어려울 수 있습니다.

### 2.5 사선(Diagonal) 적용 규칙 — 3 Rules

라인 아이콘은 T-B 로고의 사선 요소를 아이콘에 녹여내기 위해 3가지 규칙을 일관성 있게 적용합니다. 우선순위는 **Rule 1 > Rule 2 > Rule 3** 순이며, 불안정한 형태 및 디자인 완성도가 저해되는 경우 무리하게 적용하지 않습니다.

**Rule 1 — 직선에서 사선이 한 라인으로 연결되어 형태를 완성하는 방식**
- 주요 아이덴티티와 가장 부합하는 방식
- 우선 적용 대상

**Rule 2 — 일부 지점을 개방하고 직선 혹은 사선을 적용하여 형태를 완성하는 방식**
- Rule 1이 어려울 때 차선으로 적용

**Rule 3 — 일부 선의 끝지점에 사선이 위치하여 형태를 완성하는 방식**
- Rule 1, 2가 모두 어려울 때 적용

### 2.6 복수 오브젝트 조합 규칙

복수의 오브젝트를 조합해야 하는 경우, 조합되는 오브젝트는 **우측 하단에 일관성 있게 위치**시켜 전개합니다.

조형적 특징에 따라 2가지 타입 중 선택:

| 타입 | 설명 | 예시 |
|------|------|------|
| **Round** | 원형 오브젝트를 우측 하단에 배치 | N 마크, 체크 마크, 정보(i) 등 |
| **Object** | 사각 또는 비정형 오브젝트를 우측 하단에 배치 | 저장장치, 보안 실드, 수리 도구 등 |

## 3. 2D Icon — 디자인 원칙

### 3.1 디자인 철학

2D 아이콘은 **공간감과 생동감이 느껴지는 형태와 T 블루 틴티드 컬러**를 통해 브랜드 일체감을 느낄 수 있도록 제작되었습니다. **투명 배경**으로 제공되며, 다크 배경 위에 배치했을 때 최적의 가시성을 가집니다.

### 3.2 스타일 특징

| 속성 | 설명 |
|------|------|
| **색상 체계** | T 블루 계열 틴티드 컬러 (기본) + 레드 경고 컬러 (위험/차단 상태) |
| **표현 방식** | 입체감/공간감이 있는 2.5D 일러스트레이션, fill 기반 (스트로크 사용 안 함) |
| **배경** | 투명 배경(RGBA)으로 제공. 다크 배경(#000000) 위 배치 시 최적 가시성 |
| **복잡도** | Line Icon 대비 높은 디테일 |
| **용도** | 서비스 커뮤니케이션, 프로모션, 마케팅 소재, 비주얼 포인트 |
| **화이트 요소** | 아이콘 내부의 텍스트, 체크마크 등 핵심 심볼에 화이트(#FFFFFF) 사용 |

### 5.3 파일명 디코딩 규칙

PNG 파일명은 한글이 `#UXXXX` 형식으로 인코딩되어 있습니다.

```
예시:
#Uc2a4#Ub9c8#Ud2b8#Ud3f0.png → 스마트폰.png
#Uac80#Uc0c9.png → 검색.png
#Ub370#Uc774#Ud130.png → 데이터.png
```

디코딩 로직 (Python):
```python
import re

def decode_icon_name(filename):
    name = filename.replace('.png', '')
    result = ''
    i = 0
    while i < len(name):
        if name[i] == '#' and i + 1 < len(name) and name[i + 1] == 'U':
            hex_str = name[i + 2:i + 6]
            result += chr(int(hex_str, 16))
            i += 6
        else:
            result += name[i]
            i += 1
    return result
```

디코딩 로직 (JavaScript):
```javascript
function decodeIconName(filename) {
  const name = filename.replace('.png', '');
  return name.replace(/#U([0-9a-fA-F]{4})/g, (_, hex) =>
    String.fromCharCode(parseInt(hex, 16))
  );
}
```

## 6. 사용 가이드 — Do & Don't

### 6.1 Line Icon

#### ✅ Do

| 항목 | 설명 |
|------|------|
| **단색 사용** | 기본 블랙(#040000) 또는 브랜드 가이드에 따른 단색으로 사용 |
| **패딩 확보** | Total Area 기준 상하좌우 2dp의 여백을 반드시 확보 |
| **최소 사이즈 준수** | 화면 48px, 인쇄 20mm 이상에서 사용 |
| **정방형 유지** | 아이콘 비율을 임의로 변경하지 않고 1:1 비율 유지 |
| **투명 배경** | PNG 에셋의 투명 배경을 유지하여 다양한 배경 위에 배치 |
| **카테고리 맥락 준수** | 각 아이콘을 의도된 카테고리의 맥락에 맞게 사용 |
| **Rule 우선순위 준수** | 신규 아이콘 제작 시 Rule 1 → 2 → 3 순서로 적용 |

#### ❌ Don't

| 항목 | 설명 |
|------|------|
| **비율 왜곡** | 아이콘을 가로 또는 세로로 늘려서 사용하지 않음 |
| **스트로크 굵기 변경** | 0.85dp 스트로크를 임의로 굵게/얇게 변경하지 않음 |
| **멀티 컬러 적용** | 한 아이콘 내에 여러 색상을 혼용하지 않음 (단색 원칙) |
| **최소 사이즈 미달 사용** | 48px(화면) / 20mm(인쇄) 미만으로 축소하지 않음 |
| **패딩 제거** | 아이콘 주변 여백(Padding Area)을 잘라내지 않음 |
| **그림자/효과 추가** | 드롭쉐도우, 글로우 등 임의의 효과를 추가하지 않음 |
| **회전** | 아이콘을 임의로 회전시켜 사용하지 않음 |
| **Line + 2D 혼용** | 같은 UI 컨텍스트 내에서 Line과 2D 스타일을 혼용하지 않음 |
| **Rule 무리 적용** | 사선 Rule 적용 시 형태나 완성도가 저해되면 무리하게 적용하지 않음 |

### 6.2 2D Icon

#### ✅ Do

| 항목 | 설명 |
|------|------|
| **투명 배경 유지** | PNG의 투명 배경을 그대로 활용하여 별도 배경 제거 없이 사용 |
| **다크 배경 위 배치 권장** | 2D 아이콘은 어두운 배경에서 최적의 가시성을 가짐 |
| **원본 비율 유지** | AI 원본에서 내보낼 때 종횡비를 유지 |
| **충분한 크기 확보** | 입체 디테일이 보이도록 충분히 큰 사이즈에서 사용 |
| **마케팅/프로모션 맥락** | 비주얼 포인트가 필요한 마케팅 소재에 활용 |

#### ❌ Don't

| 항목 | 설명 |
|------|------|
| **라이트 배경 위 단독 사용** | 밝은 배경에서 색상 대비가 약해져 가시성이 떨어짐 (다크 카드/섹션 위에 배치할 것) |
| **배경에 불투명 색상 깔기** | 에셋 자체가 투명 배경이므로 별도 배경색을 넣어 투명도를 덮지 않음 |
| **작은 사이즈 사용** | UI 내비게이션 등 소형 아이콘 용도로 사용하지 않음 (→ Line Icon 사용) |
| **색상 변경** | T 블루 틴티드 컬러를 임의로 변경하지 않음 |
| **Line Icon과 동일 맥락 사용** | 기능 아이콘 자리에 2D 아이콘을 배치하지 않음 |

## 7. 개발 구현 가이드

### 7.1 아이콘 사이징 (CSS)

```css
/* 기본 아이콘 사이즈 체계 */
:root {
  --icon-xs: 16px;   /* 인라인 텍스트 옆 */
  --icon-sm: 24px;   /* 리스트 아이템, 버튼 내 */
  --icon-md: 32px;   /* 내비게이션, 탭 */
  --icon-lg: 48px;   /* 카드, 피처 섹션 (최소 사이즈) */
  --icon-xl: 64px;   /* 빈 상태(Empty State), 온보딩 */
  --icon-2xl: 96px;  /* 히어로 섹션 */
}

/* 아이콘 컨테이너 */
.icon-container {
  display: inline-flex;
  align-items: center;
  justify-content: center;
  width: var(--icon-md);
  height: var(--icon-md);
}

.icon-container img {
  width: 100%;
  height: 100%;
  object-fit: contain;
}
```

### 7.2 아이콘 컬러 제어 (CSS filter)

Line Icon이 PNG로 제공되므로, 다크모드 등에서 색상을 변경하려면 CSS filter를 활용합니다.

```css
/* 기본 (거의 블랙 #040000) → 그대로 사용 */
.icon-default {
  /* 필터 없음 */
}

/* 화이트로 반전 (다크모드) */
.icon-white {
  filter: invert(1);
}

/* T 블루 계열로 변경 */
.icon-brand {
  filter: invert(27%) sepia(100%) saturate(5000%) hue-rotate(240deg) brightness(90%);
}

/* 비활성 상태 (그레이) */
.icon-disabled {
  filter: invert(70%);
  opacity: 0.5;
}
```

> 💡 SVG로 변환하면 `fill` 속성으로 직접 색상을 제어할 수 있어 더 유연합니다. 가능하다면 AI 원본에서 SVG로 내보내기를 권장합니다.

### 7.3 접근성 (Accessibility)

```html
<!-- 의미 있는 아이콘: alt 텍스트 제공 -->
<img src="icons/검색.png" alt="검색" class="icon-container" />

<!-- 장식용 아이콘: 스크린리더에서 숨김 -->
<img src="icons/화살표.png" alt="" aria-hidden="true" class="icon-container" />

<!-- 버튼 내 아이콘: 버튼에 라벨 부여 -->
<button aria-label="설정">
  <img src="icons/설정.png" alt="" aria-hidden="true" />
</button>
```
